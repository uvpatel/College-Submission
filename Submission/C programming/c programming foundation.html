  • Domain Errors: If a function receives an invalid input (e.g., sqrt(-1), log(0)), it typically returns NaN (Not a Number) and sets errno to EDOM.
  • Range Errors: If the result is too large/small (e.g., overflow/underflow), it may return HUGE_VAL (or -HUGE_VAL) and set errno to ERANGE.
  • Use isnan(x), isfinite(x), or isinf(x) to check for special values (NaN, infinity).

Example:

c
#include <stdio.h> #include <math.h> #include <errno.h> int main() { double x = -1.0; double result = sqrt(x); if (errno == EDOM) { printf("Domain error: sqrt of negative number\n"); } else if (isnan(result)) { printf("Result is NaN\n"); } else { printf("Square root of %.1f = %.4f\n", x, result); } return 0; }

Output:

text
Domain error: sqrt of negative number

Tips for Using <math.h>

  1. Always Link the Math Library: Use -lm when compiling with gcc or clang.
  2. Check for Errors: Especially for functions like log, sqrt, or division-related operations.
  3. Use Radians for Trigonometric Functions: Convert degrees to radians when necessary.
  4. Floating-Point Precision: Be aware of precision issues with double or float—small errors can accumulate.
  5. Portability: Fixed-width types (e.g., float, double) behave consistently, but the exact precision or performance may vary by platform.
